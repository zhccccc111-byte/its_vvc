## 🔧 阶段1：LFNST模块重写（P0级，2-3天）

### 1.1 问题分析

**当前LFNST模块的严重问题：**
- `lfnst_tr_set_idx` 未使用（接口有但未接入逻辑）
- `nTrs` 定义错误（使用nonZeroSize=8/16，应为`(w>=8 && h>=8) ? 48 : 16`）
- 矩阵维度错误（当前8×8/16×8，应为16×16/48×16）
- ROM条目数严重不足（当前384条，需要8192条）
- 场景覆盖不足（当前4个，需要16个）

#### 任务4.2：关键路径优化
**时间**：1天
**步骤**：

1. **LFNST写地址路径优化**（最大瓶颈）：
   ```verilog
   // 当前：9级逻辑，6.975ns
   // 优化：增加流水线级数

   // 原始代码
   wire [11:0] lfnst_wr_mem_addr = lfnst_ntrs_is_48 ?
                   ({6'd0, lfnst_row48} * {5'd0, tu_width} + {9'd0, lfnst_col48}) :
                   {6'd0, lfnst_wr_addr};

   // 优化后：拆分为3级流水线
   reg [11:0] lfnst_wr_mem_addr_p1;
   reg [11:0] lfnst_wr_mem_addr_p2;
   reg [11:0] lfnst_wr_mem_addr_p3;

   always @(posedge clk) begin
       // Stage 1: 计算行偏移
       lfnst_wr_mem_addr_p1 <= lfnst_ntrs_is_48 ?
           {6'd0, lfnst_row48} * {5'd0, tu_width} : 12'd0;

       // Stage 2: 计算列偏移
       lfnst_wr_mem_addr_p2 <= lfnst_ntrs_is_48 ?
           {9'd0, lfnst_col48} : {6'd0, lfnst_wr_addr};

       // Stage 3: 累加
       lfnst_wr_mem_addr_p3 <= lfnst_wr_mem_addr_p1 + lfnst_wr_mem_addr_p2;
   end
   ```

2. **变换引擎优化**：
   - 增加MAC流水线级数
   - 优化ROM读取路径
   - 减少组合逻辑深度

3. **输出路径优化**：
   - 增加输出寄存器级数
   - 优化BRAM读取时序
   - 减少输出组合逻辑

## ⚠️ 风险与应对

### 风险1：LFNST矩阵解析困难
**概率**：中
**影响**：高
**应对**：
- 提前下载附件，尽早开始解析
- 如有疑问，邮件咨询专家
- 准备备用方案（手动生成部分矩阵）

### 风险2：500MHz时序难以满足
**概率**：高
**影响**：中
**应对**：
- 考虑使用更快的FPGA器件（UltraScale+）
- 接受较低频率（如300-400MHz），优化PPA
- 重点优化关键路径

### 风险3：数值验证发现问题
**概率**：中
**影响**：高
**应对**：
- 尽早建立黄金模型
- 逐步验证，及时发现问题
- 预留调试时间

### 风险4：时间不足
**概率**：中
**影响**：高
**应对**：
- 优先完成P0任务
- 简化P1任务
- 接受部分妥协（如降低频率目标）
