## 3. 时序分析

### 3.1 时序概览

| 指标 | 值 | 说明 |
|------|-----|------|
| WNS (Worst Negative Slack) | -8.676 ns | Setup 违例 |
| TNS (Total Negative Slack) | -104,293.805 ns | 总 Setup 违例 |
| WHS (Worst Hold Slack) | -1.046 ns | Hold 违例 |
| WPWS (Worst Pulse Width Slack) | -0.576 ns | 脉冲宽度违例 |

**结论：** 时序不满足 500MHz 约束。

### 3.2 时序违例分析

**关键路径：** out_cnt_reg → out_mem (分布式 RAM) → MUXF7/F8 → OBUF

**Setup 违例原因：**
1. **分布式 RAM 读取路径**：out_mem 有 4 个同时组合读端口，被综合为分布式 RAM，导致长组合路径
2. **输出 MUX 链**：40-bit 输出经过多级 MUX 选择

**实际可行频率估算：**
- 关键路径延迟 ≈ 2ns + 8.676ns = 10.676ns
- 可行频率 ≈ 94 MHz

## 5. 资源优化总结

### 5.1 已完成的优化

| 优化措施 | 效果 |
|---------|------|
| RAM 数组异步复位 → 无复位 (同步写入) | LUT 从 82K 降至 11K，寄存器从 192K 降至 2.5K |
| RAM 写控制逻辑分离 | 控制寄存器保留异步复位，RAM 写块去掉复位 |
| in_mem 写端口合并 | 修复双端口写入综合错误 |

### 5.2 进一步优化方向

| 优化方向 | 预期效果 | 难度 |
|---------|---------|------|
| 减少 out_mem 同时读端口数 | 推断为 Block RAM，LUT 减少 ~7000 | 高 |
| 添加输出流水线寄存器 | 改善关键路径延迟 | 中 |
| 行/列引擎 ROM 复用 | Block RAM 减少 1 个 | 低 |
| 时钟门控 | 功耗降低 ~20% | 低 |
