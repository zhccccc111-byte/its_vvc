# ITS VVC 反变换模块 — 设计文档

## 1. 概述

### 1.1 设计目标

实现 VVC (H.266) 视频编码标准的反变换模块 (Inverse Transform Subsystem, ITS)，支持：
- 三种变换核：DCT2、DCT8、DST7
- 块大小：4x4 ~ 64x64 (DCT2)，4x4 ~ 32x32 (DCT8/DST7)
- LFNST (Low-Frequency Non-Separable Transform)
- 目标时钟频率：赛题要求 500MHz（UltraScale+ FPGA 已达标，Artix-7 不可达）
- 每周期 4 点并行计算

### 1.2 标准参考

- VVC 标准：ITU-T H.266 / ISO/IEC 23090-3 (JVET-S2001)
- LFNST 矩阵：赛题附件 "Low frequency non.docx"
- 赛题规范：第九届中国研究生创芯大赛 · 华为赛题一（ITS 接口定义、500MHz 目标频率）

## 2. 架构设计

### 2.1 顶层架构

```
                    ┌─────────────────────────────────────────────────┐
                    │                   its_top                       │
                    │                                                 │
  it_info ─────────→│  ┌─────────┐                                    │
  it_data_in ──────→│  │ in_mem  │──→ ┌──────────────┐               │
  it_data_addr ────→│  │ (4096)  │    │ Row Engine   │──→ tp_buf     │
                    │  └─────────┘    │ (4 MAC)      │    (4096)     │
                    │                 └──────────────┘       │        │
                    │  ┌─────────┐                    ┌──────┘        │
                    │  │ LFNST   │                    ▼               │
                    │  │ Module  │           ┌──────────────┐         │
                    │  └─────────┘           │ Col Engine   │──→ out_mem
                    │                        │ (4 MAC)      │    (4096) │
                    │                        └──────────────┘    │     │
                    │                                            ▼     │
                    │                                     ┌──────────┐ │
                    │                                     │ Output   │ │
                    │                                     │ Control  │─→ it_data_out
                    │                                     └──────────┘ │
                    └─────────────────────────────────────────────────┘
```

### 2.2 状态机

```
S_IDLE ──→ S_LOAD ──→ [S_LFNST] ──→ S_ROW_START ──→ S_ROW_RUN ──┐
                        (可选)         (行变换启动)               │
                                                ┌────────────────┘
                                                ▼ (循环 height 次)
                                          S_COL_START ──→ S_COL_RUN ──┐
                                               (列变换启动)           │
                                                ┌─────────────────────┘
                                                ▼ (循环 width 次)
                                          S_OUT ──→ S_DONE ──→ S_IDLE
                                        (光栅输出)
```

**状态转移条件：**

| 当前状态 | 下一状态 | 条件 |
|---------|---------|------|
| S_IDLE | S_LOAD | it_info_vld 脉冲 |
| S_LOAD | S_LFNST | it_data_end 脉冲 && lfnst_idx != 0 |
| S_LOAD | S_ROW_START | it_data_end 脉冲 && lfnst_idx == 0 |
| S_LFNST | S_ROW_START | lfnst_done 脉冲 |
| S_ROW_START | S_ROW_RUN | 无条件（1周期） |
| S_ROW_RUN | S_COL_START | row_done && row_idx+1 >= height |
| S_ROW_RUN | S_ROW_START | row_done && row_idx+1 < height |
| S_COL_START | S_COL_RUN | 无条件（1周期） |
| S_COL_RUN | S_OUT | col_done && col_idx+1 >= width |
| S_COL_RUN | S_COL_START | col_done && col_idx+1 < width |
| S_OUT | S_DONE | out_cnt >= total_points |
| S_DONE | S_IDLE | 无条件（1周期） |

**输入结束检测：** 使用 `it_data_end` 信号（赛题 4/24 更新要求）。外部在发送完最后一个输入数据后，拉高 `it_data_end` 一拍。

### 2.3 数据流

1. **输入阶段 (S_LOAD)**：外部按地址写入 in_mem[0..4095]，只传非零系数
2. **LFNST 阶段 (S_LFNST)**：读取 in_mem 左上 4x4，经 LFNST 变换后写回
3. **行变换阶段 (S_ROW_RUN)**：逐行读取 in_mem，1D 变换后写入 tp_buf（行主序）
4. **列变换阶段 (S_COL_RUN)**：按步长 width 读取 tp_buf（列数据），1D 变换后写入 out_mem
5. **输出阶段 (S_OUT)**：按光栅扫描顺序每周期输出 4 个 10-bit 结果

## 3. 模块详细设计

### 3.1 变换引擎 (`its_transform_engine.v`)

执行一维反变换 `y = T^T * x`。

**端口列表：**

| 信号 | 位宽 | 方向 | 说明 |
|------|------|------|------|
| clk | 1 | I | 时钟 |
| rst_n | 1 | I | 异步复位 |
| start | 1 | I | 启动脉冲 |
| tr_type | 2 | I | 变换类型 (0=DCT2, 1=DCT8, 2=DST7) |
| size | 7 | I | 变换大小 (4/8/16/32/64) |
| data_in | 16 | I | 输入数据 |
| data_in_vld | 1 | I | 输入有效 |
| data_in_req | 1 | O | 输入请求 |
| rom_addr | 14 | O | ROM 地址 |
| rom_coeff | 16 | I | ROM 系数 |
| data_out | 16 | O | 输出数据 |
| data_out_vld | 1 | O | 输出有效 |
| data_out_req | 1 | I | 输出反压 |
| done | 1 | O | 完成脉冲 |

**内部架构：**

```
                ┌─────────────────────────────────────────┐
                │          its_transform_engine            │
                │                                         │
  data_in ─────→│  line_buf[0..63]                        │
                │       │                                 │
                │       ▼                                 │
                │  ┌─────────┐  ┌─────────┐              │
                │  │ MAC 0   │  │ MAC 1   │  ... (x4)    │
                │  │ a*b→acc │  │ a*b→acc │              │
                │  └─────────┘  └─────────┘              │
                │       │           │                     │
                │       ▼           ▼                     │
                │  result_buf[0..63]                      │
                │       │                                 │
                │       ▼                                 │
  data_out ←────│  (sum + 32) >>> 6                       │
                └─────────────────────────────────────────┘
```

**计算流程（N 点变换）：**

1. S_LOAD (N 周期)：加载 N 个输入到 line_buf
2. S_PREFETCH (4N 周期)：从 ROM 预取 4 行系数到 coeff_buf
3. S_COMPUTE (N 周期)：4 MAC 并行计算 4 个输出点
4. 若 N > 4，需 N/4 组，重复步骤 2-3
5. S_OUTPUT (N 周期)：输出结果

**行分组处理：** 对于 N > 4 的变换，将 N 行分成 N/4 组，每组 4 行共享同一次 ROM 预取。row_group 计数器跟踪当前组。

### 3.2 MAC 单元 (`its_mac.v`)

2 级流水线乘累加器。

**流水线时序：**

```
周期 0: en=1 → product = a * b (Stage 1)
周期 1: valid_s1=1 → result += sign_ext(product) (Stage 2)
周期 2: valid=1 (结果可用)
```

**清零机制：** `clr` 信号优先级高于累加，用于每组计算开始前复位累加器。

**位宽：**
- 输入 a, b: 16-bit signed
- 乘积 product: 32-bit signed
- 累加器 result: 40-bit signed

### 3.3 LFNST 模块 (`its_lfnst.v`)

执行 LFNST 反变换。

**公式：** `y[i] = clip3(-32768, 32767, (Σ_j T[i][j]·x[j] + 64) >> 7)`

**nTrs 定义：**
- `nTrs = (tu_width >= 8 && tu_height >= 8) ? 48 : 16`
- nTrs=16: 16x16 矩阵，16 输入 → 16 输出
- nTrs=48: 48x16 矩阵，16 输入 → 48 输出

**状态机：**

```
S_IDLE → S_LOAD → S_PREFETCH → S_COMPUTE → S_DRAIN → S_OUTPUT → S_DONE
```

**计算流程：**

1. S_LOAD：加载 16 个输入到 in_buf
2. S_PREFETCH：从 LFNST ROM 预取 nTrs*16 个系数到 coeff_buf
3. S_COMPUTE：16 周期内积计算（MAC 串行）
4. S_DRAIN：等待 MAC 流水线排空（2 周期）
5. S_OUTPUT：裁剪并输出结果（nTrs 个点）
6. S_DONE：完成

**ROM 地址布局（8192 条目）：**

```
nTrs=16 [0..2047]:
  base = lfnstTrSetIdx * 512 + (lfnst_idx-1) * 256
  每场景: 16x16 = 256 条

nTrs=48 [2048..8191]:
  base = 2048 + (lfnstTrSetIdx * 2 + (lfnst_idx-1)) * 768
  每场景: 48x16 = 768 条
```

### 3.4 顶层 LFNST 集成

**读地址映射（nTrs=48）：**

LFNST 读取 in_mem 左上 4x4 子块（16 个元素），地址计算：
```
row = rd_addr[3:2]
col = rd_addr[1:0]
addr = row * tu_width + col
```

**写回地址映射：**

nTrs=16：顺序写回 in_mem[0..15]

nTrs=48：3 个 4x4 子块布局：
```
blk 0: rows 0-3, cols 0-3  → addr = row * width + col
blk 1: rows 0-3, cols 4-7  → addr = row * width + (col+4)
blk 2: rows 4-7, cols 0-3  → addr = (row+4) * width + col
```

## 4. ROM 结构

### 4.1 变换核 ROM (`its_rom.v`)

8176 条目，16-bit 系数，同步读（1 周期延迟）。

| 变换类型 | 块大小 | 起始地址 | 系数数量 |
|---------|--------|---------|---------|
| DCT2 | 4 | 0 | 16 |
| DCT2 | 8 | 16 | 64 |
| DCT2 | 16 | 80 | 256 |
| DCT2 | 32 | 336 | 1024 |
| DCT2 | 64 | 1360 | 4096 |
| DCT8 | 4 | 5456 | 16 |
| DCT8 | 8 | 5472 | 64 |
| DCT8 | 16 | 5536 | 256 |
| DCT8 | 32 | 5792 | 1024 |
| DST7 | 4 | 6816 | 16 |
| DST7 | 8 | 6832 | 64 |
| DST7 | 16 | 6896 | 256 |
| DST7 | 32 | 7152 | 1024 |

**地址公式：** `addr = base_addr + row * size + col`

**系数生成公式：**
- DCT2: T(0,j) = 64, T(i,j) = round(89 * cos(π·i·(2j+1)/(2N))) for i>0
- DCT8: T(i,j) = round(64 * cos(π·(2i+1)·(2j+1)/(4N)))
- DST7: T(i,j) = round(64 * sin(π·(i+1)·(j+1)/(N+1)))

### 4.2 LFNST ROM (`its_lfnst_rom.v`)

8192 条目，16-bit 系数，同步读（1 周期延迟）。

ROM 地址布局：
- [0..2047]：nTrs=16 (8 场景 × 256 条)
- [2048..8191]：nTrs=48 (8 场景 × 768 条)

## 5. 定点量化方案

### 5.1 量化公式

| 模块 | 运算 | 量化方式 |
|------|------|---------|
| 变换引擎 | y = T * x | (sum + 32) >>> 6 |
| LFNST | y = T * x | (sum + 64) >>> 7，clip3(-32768, 32767) |
| 输出 | 40-bit → 10-bit | 截取低 10 位 |

**位宽分析：**
- 输入系数: 16-bit signed [-32768, 32767]
- 变换核系数: 16-bit signed
- MAC 累加器: 40-bit signed（避免 16×16×N 溢出）
- 输出结果: 10-bit signed [-512, 511]

### 5.2 详细量化定标分析

#### 5.2.1 数据格式定义

| 信号 | 格式 | 范围 | 说明 |
|------|------|------|------|
| 输入系数 x[j] | Q0 (整数) | [-32768, 32767] | 视频解码器输出的频域系数 |
| 变换核 T[i][j] | Q0 (整数) | [-89, 89] | DCT2/DCT8/DST7 整数基矩阵 |
| LFNST 矩阵 | Q0 (整数) | [-128, 127] | 官方 LFNST 变换矩阵 |
| MAC 乘积 | Q0 | [-2916352, 2916352] | 16-bit × 16-bit = 32-bit |
| MAC 累加和 | Q0 | 见下文分析 | 40-bit 有符号累加器 |
| 变换引擎输出 | Q0 | [-32768, 32767] | (sum + 32) >>> 6 |
| LFNST 输出 | Q0 | [-32768, 32767] | (sum + 64) >>> 7，clip |
| 最终输出 | Q0 | [-512, 511] | 10-bit 有符号 |

#### 5.2.2 溢出分析

**MAC 累加器位宽需求：**

变换引擎计算 `y[i] = (Σ_j T[i][j]·x[j] + 32) >> 6`。

最坏情况分析（所有输入取最大值，系数取最大值）：

| 变换类型 | 最大 |T[i][j]| | 最大 |x[j]| | 最大乘积 | N (最大块) | 最大累加和 | 所需位宽 |
|---------|---------|---------|---------|---------|---------|---------|
| DCT2 | 89 | 32768 | 2,916,352 | 64 | 186,646,528 | 28 bit |
| DCT8 | 64 | 32768 | 2,097,152 | 32 | 67,108,864 | 27 bit |
| DST7 | 64 | 32768 | 2,097,152 | 32 | 67,108,864 | 27 bit |
| LFNST | 128 | 32768 | 4,194,304 | 16 | 67,108,864 | 27 bit |

40-bit 累加器（有效范围 2^39 = 549,755,813,887）远超最大累加和，**无溢出风险**。

**中间结果位宽：**

行变换输出经 `>>> 6` 后为 16-bit，作为列变换输入。列变换再次执行相同运算：
- 列变换最大累加和 = 64 × 89 × 32767 = 186,640,832 (28 bit)
- `>>> 6` 后最大值 = 186,640,832 / 64 ≈ 2,916,263 (22 bit)
- 但输出截取为 10-bit [-512, 511]，符合 VVC 标准残差范围

#### 5.2.3 量化误差分析

**变换引擎 `>>> 6` 误差：**

`>>> 6` 等价于除以 64 并向下取整。添加 `+32`（即 64/2）实现四舍五入：
- 理想值: y_exact = sum / 64
- 实际值: y_actual = (sum + 32) >> 6
- 最大量化误差: |y_exact - y_actual| ≤ 0.5 LSB

**LFNST `>>> 7` 误差：**

同理，`+64` 实现四舍五入：
- 最大量化误差: |y_exact - y_actual| ≤ 0.5 LSB

**2D 变换误差累积：**

2D 反变换 = 列变换(行变换(x))。两次 `>>> 6` 的误差累积：
- 行变换误差: ≤ 0.5 LSB (行变换输出域)
- 列变换误差: ≤ 0.5 LSB (最终输出域)
- 总误差: ≤ 1.0 LSB (最终输出域)

由于输出为 10-bit [-512, 511]，1.0 LSB 的误差相对于信号范围极小（< 0.2%），对视频重建质量影响可忽略。

#### 5.2.4 与浮点模型对比

VVC 标准的浮点反变换公式：
```
y[i] = Σ_j T[i][j] · x[j]    (浮点)
```

本设计的定点实现：
```
y[i] = (Σ_j T[i][j] · x[j] + 32) >>> 6    (定点)
```

**误差来源：**
1. 变换核系数量化：浮点 cos/sin 值被量化为 7-bit 整数（-89 ~ 89），引入 ≤ 0.5 的系数误差
2. 右移舍入：`>>> 6` 引入 ≤ 0.5 LSB 的舍入误差
3. 输出截取：10-bit 输出截取引入 ≤ 0.5 LSB 的截取误差

**精度评估：**
- 对于典型视频残差信号（幅度 < 100），量化误差 < 1%
- 对于极端情况（幅度接近 512），量化误差 < 0.2%
- 整体 PSNR 损失 < 0.1 dB（可忽略）

#### 5.2.5 LFNST 精度分析

LFNST 使用 16-bit 有符号整数矩阵，变换公式：
```
y[i] = clip3(-32768, 32767, (Σ_j T[i][j]·x[j] + 64) >>> 7)
```

- 输入: 16-bit 系数（来自视频解码器或 LFNST 前的变换）
- 矩阵值: 16-bit 整数（官方文档精确值）
- 累加器: 40-bit（16 个乘积累加，最大 27 bit，无溢出）
- 右移 7 位: 等效除以 128，+64 实现四舍五入
- clip: 确保输出在 16-bit 范围内

LFNST 的精度完全由官方矩阵系数决定，本设计使用与标准一致的整数矩阵，**精度无损**。

## 8. 目标频率分析

### 8.1 赛题要求

赛题要求目标时钟频率 500MHz（2ns 周期）。

### 8.2 UltraScale+ 实现结果 (v4.0)

v3.9 的 RTL 在 Kintex UltraScale+ xcku5p-ffvb676-2-e 上零改动即满足 500MHz：

| 指标 | 值 | 状态 |
|------|-----|------|
| WNS (Setup) | **+0.030 ns** | **MET** |
| TNS | 0.000 ns | — |
| WHS (Hold) | +0.020 ns | MET |
| Failing Endpoints | **0** | — |

**Worst Path**: LFNST ROM→DistRAM（0 级逻辑，1.846ns），不再是 DSP 路径。

**为什么 UltraScale+ 能达标**: DSP48E2 相比 DSP48E1 大幅改善了 FF propagation 和 setup time，消除了 Artix-7 上 DSP48E1 FF→A 路径的固有瓶颈。Block RAM (RAMB36E2) 的最小周期也从 2.234ns 降至约 1.8ns，消除了 BRAM 硬约束。

### 8.3 Artix-7 实现限制

500MHz 目标在 Artix-7 xc7a200tfbg484-3 上**物理不可达**（v3.9 WNS = -1.733ns）。

**不可行原因：**

| 延迟组件 | 值 | 可优化性 | 说明 |
|---------|-----|---------|------|
| DSP48E1 FF→A propagation | 0.341 ns | 不可优化 | DSP 硅片物理特性 |
| FF→DSP 路由 | ~0.58 ns | 部分可优化 | 已从 0.679ns 优化到 0.534ns |
| DSP48E1 setup time | 0.106 ns | 不可优化 | DSP 硅片物理特性 |
| 时钟偏移/不确定性 | ~0.07 ns | 不可优化 | OOC 模式 |
| **WNS 合计** | **-1.733 ns** | | 对应最高 ~362 MHz |

v3.3→v3.9 的 RTL 优化（+0.556ns）消除了全部逻辑瓶颈，剩余 gap 全部来自 DSP48E1 固有物理特性。

### 8.4 ASIC 实现可行性

500MHz 目标在 ASIC 工艺下**可行**：

| 参数 | FPGA (Artix-7) | FPGA (UltraScale+) | ASIC (28nm) |
|------|----------------|---------------------|-------------|
| DSP FF→A | 0.341 ns | ~0.15 ns | ~0.05 ns |
| SRAM 读取 | 2.125 ns | ~0.96 ns | ~0.5 ns |
| 时钟偏斜 | 3.920 ns | ~0.04 ns | < 0.5 ns |
| 预估最高频率 | ~96 MHz | **500 MHz** | ~500 MHz |

### 8.5 结论

- **UltraScale+**: 500MHz **已达标**（WNS +0.030ns），相同 RTL 零改动
- **Artix-7**: 500MHz 不可达（WNS -1.733ns），受限于 DSP48E1 固有物理特性
- **ASIC**: 500MHz 可行，设计架构已针对高频优化（4 MAC 并行、流水线、寄存器输出）
